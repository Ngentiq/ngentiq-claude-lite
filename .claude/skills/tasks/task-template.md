# Task Output Template

This is the output format for task files generated by `/tasks`. Each task is written to `plans/P-{NNN}-{slug}/T-{NNN}.md`.

---

```yaml
---
task: T-NNN
plan: P-NNN
status: pending
depends-on: []
effort: small
satisfies-bac: [BAC-1]
satisfies-qac: [QAC-1, QAC-2]
---
```

```markdown
# T-NNN: {Task Title}

## Description

{What this task accomplishes and why it is needed. 2-4 sentences. Reference the plan requirements this maps to.}

## Scope

| Item | Detail |
|------|--------|
| Files to create | {list or "none"} |
| Files to modify | {list or "none"} |
| Files to delete | {list or "none"} |

## Acceptance Criteria

### Business

- [ ] BAC-N: {criterion text from plan, verbatim}
- [ ] BAC-N: {criterion text from plan, verbatim}

### QA

- [ ] QAC-N: {criterion text from plan, verbatim}
- [ ] QAC-N: {criterion text from plan, verbatim}

List only the BAC/QAC items this specific task satisfies.
Copy the criterion text verbatim from the plan to maintain traceability.

## Notes

{Implementation hints, patterns to follow from existing code, gotchas, or references to related tasks. Omit section if empty.}
```

## Frontmatter Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `task` | string | Task identifier (T-001 through T-NNN) |
| `plan` | string | Parent plan identifier (P-NNN) |
| `status` | enum | `pending`, `in-progress`, `done`, `failed`, `skipped` |
| `depends-on` | list | Task IDs that must complete before this task starts |
| `effort` | enum | Estimated effort: `trivial`, `small`, `medium`, `large` |
| `satisfies-bac` | list | BAC item IDs this task satisfies |
| `satisfies-qac` | list | QAC item IDs this task satisfies |
